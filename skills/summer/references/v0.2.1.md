# Summer Framework — v0.2.1

Released: 2026-02-26

## Breaking Changes from pre-0.2.1

This was a major restructuring release. If working with a project on 0.2.1, these are the key differences from earlier
versions:

### Tracing: Custom infra removed, Micrometer + OpenTelemetry replaces it

**Deleted:** `TraceIdMdcFilter`, `LoggingConfig`, `LoggingConfig.MDCUtils`, `RestLoggingConfig`, `LogType`, `Logging`.
**Deleted config:** `f8a.common.logging.mdc.*`, `mdc-enabled`, `service-name`.

**Required dependencies:**

```gradle
implementation 'io.micrometer:micrometer-tracing-bridge-otel'
implementation 'io.opentelemetry:opentelemetry-exporter-otlp'
```

**Required config:**

```yaml
spring:
  reactor:
    context-propagation: auto
management:
  tracing:
    sampling:
      probability: 1.0
```

`traceId` is now automatically in MDC via Micrometer — no manual filter needed. See `docs/HOW_WE_TRACE.md`.

### Security modules restructured

- `summer-security-core` renamed to `summer-jwt-resource-server` (artifact: `summer-jwt-resource-server`)
- All classes in `io.f8a.summer.security.common.*` and `io.f8a.summer.security.jwt.*` moved to new module
- New module `summer-apikey-resource-server` — API key auth classes moved here from old `security-core`
- **Deleted:** `ReactiveSecurityProperties`, `ApiKeyAuthentication`, `UserinfoServerAuthenticationEntryPoint`
- **Added:** `SecurityErrorResponseWriter` in `jwt-resource-server`
- `ReactiveApisixCustomizer` now takes `ObjectMapper` as constructor parameter
- `UserinfoGrantedAuthoritiesConverter.authoritiesClaimName` changed from field to protected method

**Migration:** Change `summer-security-core` dependency to `summer-jwt-resource-server`. If using API key auth, add
`summer-apikey-resource-server`.

### Exception handling restructured

- `GlobalExceptionHandler` renamed to `SummerGlobalExceptionHandler`
- Implements `ErrorWebExceptionHandler` (was `WebExceptionHandler`)
- Reads `traceId` from MDC directly (no `TraceIdMdcFilter`)
- Rewritten with Java 21 `switch` pattern matching
- Now handles: `AccessDeniedException`, `AuthenticationException`, `NoResourceFoundException`,
  `MethodNotAllowedException`, `NotAcceptableStatusException`, `UnsupportedMediaTypeStatusException`,
  `PayloadTooLargeException`, `IllegalArgumentException`, `IllegalStateException`
- `DownstreamException` maps to `500` (was `502`)
- `JsonErrorResponse` moved from `rest-common.exception` to `core.exception`
- `JsonErrorResponse` gains `timestamp` (Instant) and `details` (List<FieldErrorDetail>) fields
- `ViewableException` gains `details` list and fluent `.detail(field, issue)` builder
- New `CommonExceptions` entries: `NOT_ACCEPTABLE`, `UNSUPPORTED_MEDIA_TYPE`, `PAYLOAD_TOO_LARGE`

### Audit API changes

- `auditNonEntity` parameter order changed: `(intent, action, comment)` -> `(action, intent, comment)`
- `auditCustom(...)` deprecated in favor of `audit(AuditLog)` with builder
- `@Audit` on Flux now audits once after completion (was once per element — bug fix)
- `@Audit` on Mono fixed double-subscribe issue
- Warning log added when `@Audit` is applied to non-reactive methods

### Removed config properties

- `f8a.outbox.validate-schema` removed (auto-detected via `SELECT current_schema()`)
- `f8a.audit.validate-schema` removed (auto-detected)

### Deleted utilities

- `JsonUtils` (173 lines in `core.json`) — deleted entirely
- `BusinessChange` model (`core.audit.model`) — deleted
- `ContextConstants.TRACE_ID_KEY` — deleted
- Embedded Flyway migration scripts from `data-audit` — deleted

## New Features in 0.2.1

- `AbstractTableValidator` base class for startup schema validation
- `AuditTableValidator` — validates `audit_log` table (23 columns) on startup
- `OutboxTableValidator` — enhanced with column-type validation (9 columns)
- `audit(AuditLog)` method — build custom audit log with builder, auto-fills nulls
- `SecurityErrorResponseWriter` — writes `JsonErrorResponse` for security filter errors
- `DefaultAuthenticationException` and `DefaultJwtConverter` in `jwt-resource-server`

## Version-Specific Config

Role sync in 0.2.1 uses `sync-role.enable` (not `enabled`):

```yaml
f8a:
  security:
    apisix:
      resource-server:
        sync-role:
          enable: true  # NOTE: renamed to "enabled" in 0.2.3
          server-url: https://keycloak.example.com
          realm: master
          client-id: admin-cli
          client-secret: secret
```
