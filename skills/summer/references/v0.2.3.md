# Summer Framework — v0.2.3

Released: 2026-03-09

## What's New

No breaking changes from 0.2.2 (property rename is backwards-compatible).

### KeycloakException error mapping expanded (~5 -> ~58 cases)

Comprehensive auto-mapping of Keycloak error responses covering: auth, user state, token lifecycle, session, client
credentials, OAuth parameters, PKCE, and identity provider errors.

Each case maps to a typed error code with appropriate HTTP status. Falls back to `keycloak.<errorDescription>` (500) for
unknown errors.

See `references/api_reference.md` → **KeycloakException** for the full error mapping table (added in this version).

### Role sync enabled by default

- `matchIfMissing = true` on `sync-role.enabled` property — activates whenever `sync-role.server-url` is non-blank
- No need for explicit `enabled: true` in config
- Disable with `sync-role.enabled: false`

### Property renamed: `sync-role.enable` -> `sync-role.enabled`

The old `enable` property no longer works. Update your config:

```yaml
# Before (0.2.1, 0.2.2)
sync-role:
  enable: true

# After (0.2.3+)
sync-role:
  enabled: true  # or just omit — defaults to true
```

### Role sync activation condition

The auto-configuration uses `@ConditionalOnExpression` with `StringUtils.hasText()` to check `server-url`:

- `null` / empty / blank -> sync disabled
- Non-blank value -> sync enabled (unless `enabled: false`)

This means an empty `server-url:` in YAML won't accidentally trigger the synchronizer.

`RoleDefinitionScanner` is now registered as a Spring bean by the auto-configuration (available for injection without
role sync being active).

### Role sync config (0.2.3)

```yaml
f8a:
  security:
    apisix:
      resource-server:
        enabled: true        # default true (matchIfMissing)
        sync-role:
          # enabled: true    # default true — omit or set false to disable
          server-url: https://keycloak.example.com
          realm: master
          client-id: admin-cli
          client-secret: secret
          client-defaults:
            service-accounts-enabled: true
            public-client: false
            protocol: openid-connect
```
